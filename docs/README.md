# Site

Static portfolio site for the course individual project.

This folder is the canonical GitHub Pages publish directory.

## Related folders

- `../design/`: planning notes, positioning, project cards, and site design materials

## Structure

- `index.html`: main one-page portfolio
- `styles.css`: site styles
- `assets/images/`: site images
- `assets/resume/`: downloadable resume files
- `.nojekyll`: disables Jekyll processing on GitHub Pages

## GitHub Pages publishing

Recommended configuration:

1. Push the repository to GitHub.
2. Open repository `Settings`.
3. Open `Pages`.
4. Set:
   - `Source`: `Deploy from a branch`
   - `Branch`: main
   - `Folder`: `/docs`

Alternative options:

- publish from a dedicated `gh-pages` branch
- move the site to the repository root and publish from `/root`

## Before final submission

- Replace the current DOCX resume with a polished PDF if possible.
- Add 1-2 project screenshots for stronger evidence.
- Double-check all external repository links.
- Test the site on mobile width before submission.
- Verify the final public URL works before exporting the Word submission file.
