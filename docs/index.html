Personal Project Jan 2026 - Mar 2026

Chimera-Iceclaw

A secure and durable AI agent control plane for multi-channel interaction, long-running routines, and safety-governed tool execution.

Rust Control Plane WASM Sandbox Routines Engine

Problem: Many AI assistants are effective in demo settings but weak in reliability, durable execution, and safe tool boundaries for real use.

Data: The system manages local workspace state, routine execution records, tool history, user configuration, and persisted memory with traceable runtime behavior.

Approach: I designed a Rust-based control plane with multi-channel access, long-running routines, parallel job handling, a web gateway, WASM sandboxing, allowlisting, secret protection, and layered safety checks.

Outcome / Impact: The result is a production-oriented agent architecture with clear boundaries across channels, scheduling, orchestration, tools, and safety. The codebase includes extensive automated tests and explicit end-to-end quality planning.

My Contribution: Personal project. I owned the architecture direction, control-plane model, safety design, and the core runtime abstractions that make durable AI workflows possible.

Personal Project Feb 2026 - Mar 2026

Scout.cn-Agents

A Chinese intelligence-agent system that turns fragmented source data into structured monitoring, alerting, and action workflows.

Intelligence Ingestion Monitoring TypeScript Migration

Problem: Intelligence collection across Chinese platforms is fragmented, which makes it hard to convert raw signals into actionable business, monitoring, or risk insight.

Data: The system integrates multi-platform content from MediaCrawler and WeChat sources into a unified event model with incremental checkpoints, retry handling, and dead-letter queue support.

Approach: I defined the architecture across ingestion, normalization, analysis, and action layers, and designed a staged migration path from Python-based crawlers to a TypeScript control plane with scheduler and monitor APIs.

Outcome / Impact: The project delivers a working pipeline from collection to alerting, with monitor endpoints, scheduler logic, and runbook-oriented structure for practical monitoring use.

My Contribution: Personal project. I defined the architecture blueprint, the control-plane direction, the event model, and the operational logic that turns scattered collection capability into a coherent intelligence system.

Group Project Mar 2026 - Apr 2026

EA-Lab

A Mario-like procedural content generation MVP using explicit level encoding, hard feasibility constraints, and multi-objective evolutionary search.

EA PCG Feasibility Gate Technical Lead

Problem: Generate playable Mario-like levels while controlling diversity and difficulty through an auditable optimization pipeline.

Data: The MVP uses explicit chromosome-based level representations, segment libraries, decoder rules, and structured constraint and evaluation outputs rather than relying on opaque generation.

Approach: I helped build a fixed pipeline of chromosome to decode to constraint check to evaluation to selection, with hard feasibility gates and multi-objective evolutionary search.

Outcome / Impact: A reproducible technical MVP with interface documents, run scripts, feasibility-first logic, and tests for constraints and NSGA-style selection — a stable baseline for team experimentation.

My Contribution: In this group project I took a stronger technical lead role and worked on the core pipeline and EA side, especially workflow definition, feasibility-first design, and the optimization backbone.
